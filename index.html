Professional Self-Assessment

Completing the Computer Science program and developing this ePortfolio helped me connect my existing software engineering experience with a stronger academic foundation in computer science. I returned to finish the final year of my degree after taking a long break, and the capstone became a practical way to show both technical growth and follow-through. My goal throughout the project was to demonstrate that I can take an existing application, evaluate its weaknesses, and improve it using professional software engineering practices.

My selected artifact, Lipstick Matcher, began as a personal web application built to experiment with face-api.js, facial landmark detection, and local image analysis in the browser. The original version proved the basic concept, but the architecture, recommendation logic, and data storage approach needed stronger structure. Through the capstone enhancements, I refactored the application into clearer modules, improved the shade-matching algorithm, added automated tests, and introduced a PostgreSQL-backed shade catalog with a server-side data access layer.

The project reflects my interest in full-stack application development. On the front end, I worked with SvelteKit components, upload handling, progress states, and user-facing result presentation. In the application logic, I improved the recommendation process by using RGB, HSL, and Lab-style lightness calculations, multi-image sample aggregation, undertone detection, weighted scoring, and ranked shade results. On the back end, I added database schema and seed files, a repository layer, API filtering, and fallback behavior so the application remains usable even when a database connection is not configured.

The Computer Science program also strengthened areas beyond this artifact. Course work in software testing helped me better appreciate automated validation and the role of repeatable tests in maintaining quality. Database course work helped me think more carefully about schema design, relationships, queries, and how data should be organized for future growth. Software development lifecycle course work helped me better understand planning, communication, stakeholder expectations, and how responsibilities fit together on larger teams.

Collaboration and communication are also represented in this portfolio. The code review video explains technical decisions for an audience of peers or managers, while the written narratives describe what changed, why the changes mattered, and how each enhancement connects to the course outcomes. Those materials show my ability to communicate technical work clearly, not just complete the code itself.

Security and privacy were important considerations because Lipstick Matcher works with user-uploaded photos. I kept image analysis local to the browser rather than storing user images, used environment-based database configuration, avoided committing local secrets, validated API filter input, used parameterized database queries, and added basic security headers. These choices reflect a security mindset that considers privacy and misuse risks early in the design.

Together, the artifacts in this portfolio show growth across software design, algorithms and data structures, databases, testing, communication, and security. The enhanced Lipstick Matcher project represents the kind of work I want to continue doing professionally: building practical, maintainable, user-centered full-stack software that solves a real problem and can continue to evolve.
